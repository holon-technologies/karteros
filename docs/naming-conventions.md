# Naming Conventions for Holon Technologies

## Etymology and Foundation

The word **holon** (Ancient Greek: ὅλον) is a combination of the Greek *holos* (ὅλος) meaning 'whole', with the suffix *-on* which denotes a particle or part (as in proton and neutron). 

Holons are self-reliant units that possess a degree of independence and can handle contingencies without asking higher authorities for instructions (i.e., they have a degree of autonomy). These holons are also simultaneously subject to control from one or more of these higher authorities. The first property ensures that holons are stable forms that are able to withstand disturbances, while the latter property signifies that they are intermediate forms, providing a context for the proper functionality for the larger whole.

## Naming Philosophy

Our naming conventions should reflect the holon concept: technologies that are simultaneously autonomous and cooperative, self-contained yet interconnected, stable yet adaptive.

## Technology Naming Patterns

### Core System Names

**Pattern**: `[Greek Root] + [Descriptive Suffix]`

Examples:
- **HolonOS** - Core operating system for autonomous holons
- **HolonMesh** - Network of interconnected holons
- **DataSphere** - Distributed data management across holons

### Component and Service Names

**Pattern**: `[Greek/Latin Root describing function] + [System integration suffix]`

**Suggested Greek/Latin Roots:**
- *Oikos* (οἶκος) - dwelling, organized unit (can represent households, communities, or other organized groups)
- *Koinon* (κοινόν) - common, shared, community
- *Synodos* (σύνοδος) - assembly, meeting together
- *Archipelagos* (ἀρχιπέλαγος) - chief sea, group of islands
- *Sympatheia* (συμπάθεια) - fellow-feeling, connection
- *Autonomia* (αὐτονομία) - self-governance
- *Systema* (σύστημα) - organized whole

**Examples:**
- **OikosCore** - Core holon orchestration system
- **KoinonSync** - Shared data synchronization service
- **ArchipelagoNet** - Network of connected holon islands
- **SympathLink** - Inter-holon communication protocol
- **AutonoMesh** - Self-governing network mesh

### Application Integration Names

**Pattern**: `[Existing App] + [Bridge/Link/Mesh suffix]`

Examples:
- **ImmichBridge** - Integration layer for Immich photo service
- **NextcloudMesh** - Nextcloud integration with holon networks
- **JellyfinLink** - Media streaming coordination service

### Protocol and Standard Names

**Pattern**: `[Greek concept] + [Protocol designation]`

Examples:
- **HolonProtocol** - Core inter-holon communication standard
- **OikosAuth** - Holon authentication and identity management
- **KoinonShare** - Cross-holon data sharing protocol
- **SynodosDiscovery** - Holon network discovery protocol

## Naming Guidelines

### Do's
- **Reflect dual nature**: Choose names that suggest both independence and interconnection
- **Use meaningful roots**: Prefer Greek/Latin roots that relate to the function or philosophy
- **Maintain consistency**: Follow established patterns within each category
- **Consider pronunciation**: Ensure names are reasonably pronounceable in English
- **Suggest functionality**: Names should hint at what the technology does
- **Honor the commons**: Avoid proprietary-sounding or corporate naming patterns

### Don'ts
- **Avoid generic terms**: Don't use overused tech terms like "Cloud", "Smart", "AI" unless truly descriptive
- **Don't force Greek**: If a simple English name better serves users, prefer clarity
- **Avoid complexity**: Don't create names that are difficult to remember or spell
- **Don't conflict**: Research existing projects to avoid naming conflicts
- **Avoid ownership language**: Don't use possessive or exclusive-sounding names

## Special Considerations

### User-Facing vs Technical Names

**User-Facing**: Should prioritize clarity and approachability
- "HolonOS" (core system name that reflects philosophy)
- "Connected Network" (not "HolonMesh") 
- "Backup Circle" (not "ReplicationSphere")

**Technical/Developer-Facing**: Can use more sophisticated etymological naming
- "KoinonSync" for data synchronization protocols
- "ArchipelagoDiscovery" for network topology management
- "SympathAuth" for cross-holon authentication

### Versioning and Evolution

When technologies evolve or split:
- **Major revisions**: Consider new names that reflect changed purpose
- **Forks/Variants**: Use descriptive suffixes (HolonOS-Lite, HolonOS-Enterprise)
- **Experimental**: Use "Proto-" prefix (Proto-FamilyMesh)
